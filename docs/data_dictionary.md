# Data Dictionary — AI-Powered Energy Consumption Forecasting

## Raw Dataset: UCI Household Power Consumption

**Source:** [UCI Machine Learning Repository](https://archive.ics.uci.edu/ml/datasets/Individual+household+electric+power+consumption)  
**Period:** December 2006 – November 2010 (47 months)  
**Granularity:** 1-minute intervals  
**Records:** ~2,075,259 rows

### Column Descriptions

| Column | Type | Unit | Description |
|--------|------|------|-------------|
| `Date` | string | dd/mm/yyyy | Measurement date |
| `Time` | string | hh:mm:ss | Measurement time |
| `Global_active_power` | float | kW | Household global minute-averaged active power |
| `Global_reactive_power` | float | kW | Household global minute-averaged reactive power |
| `Voltage` | float | V | Minute-averaged voltage |
| `Global_intensity` | float | A | Household global minute-averaged current intensity |
| `Sub_metering_1` | float | Wh | Kitchen (dishwasher, oven, microwave) |
| `Sub_metering_2` | float | Wh | Laundry room (washing machine, dryer, fridge) |
| `Sub_metering_3` | float | Wh | Electric water heater + air conditioner |

> **Missing values:** ~1.25% of rows have `?` for all power/current columns — typically correspond to meter outage periods.

## Simulated Dataset: Virtual Smart Grid

**Generated by:** `src/simulator.py`  
**Granularity:** Hourly (configurable)

### Column Descriptions

| Column | Type | Description |
|--------|------|-------------|
| `Global_active_power` | float | Simulated household active power (kW) |
| `temperature_c` | float | Synthetic ambient temperature (°C) |
| `humidity_pct` | float | Synthetic relative humidity (%) |
| `occupancy` | float | Occupancy signal [0–1] (0=empty, 1=fully occupied) |
| `Sub_metering_1` | float | Kitchen energy share (Wh) |
| `Sub_metering_2` | float | Laundry energy share (Wh) |
| `Sub_metering_3` | float | HVAC energy share (Wh) |
| `is_weekend` | int | 1 if Saturday/Sunday, else 0 |
| `is_holiday` | int | 1 if public holiday (placeholder), else 0 |

## Engineered Features

| Feature | Description |
|---------|-------------|
| `hour` | Hour of day (0–23) |
| `dayofweek` | Day of week (0=Mon, 6=Sun) |
| `month` | Month (1–12) |
| `season` | 0=Winter, 1=Spring, 2=Summer, 3=Autumn |
| `is_peak_hour` | 1 if 7–10 AM or 18–22 PM |
| `hour_sin / hour_cos` | Cyclical encoding of hour |
| `month_sin / month_cos` | Cyclical encoding of month |
| `dow_sin / dow_cos` | Cyclical encoding of day-of-week |
| `Global_active_power_lag_N` | Power value N hours ago (N = 1,2,3,6,12,24,48,168) |
| `Global_active_power_roll_mean_N` | Rolling mean over past N hours |
| `Global_active_power_roll_std_N` | Rolling std over past N hours |
| `temp_x_hour` | temperature × hour interaction |
| `hdd` | Heating Degree: max(0, 18 − temp) |
| `cdd` | Cooling Degree: max(0, temp − 18) |

| File | Description |
|------|-------------|
| `outputs/forecasts/forecast_results.csv` | XGBoost/LSTM predicted vs actual |
| `outputs/metrics/model_metrics.json` | MAE, RMSE, MAPE, SMAPE, R² for all models |
| `outputs/reports/energy_report.pdf` | Auto-generated 5-section PDF report |
| `models/arima_model.pkl` | Serialised ARIMA result object |

| `models/lstm_model.h5` | TensorFlow LSTM weights |
| `models/target_scaler.pkl` | MinMaxScaler for LSTM target normalisation |
